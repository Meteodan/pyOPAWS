How to run

 

Python packages:  pyart, numpy, scipy, optparse, netCDF4, datetime, pyproj.  Works under the Anaconda2 system.

 

1.    To run the lvl-2 plotting, you can simply run the script.

2.    To run the opaws2d.py script, you must first:

a.    fcompile_cressman.py – compiles the fortran cressman routine, assumes GNU compiler is installed before the Anaconda python was installed.  Note:  “conda install libgfortran” can be helpful.

b.    Once a “cressman.so” exists and can be loaded – opaws2d is ready to run

 

To test the scripts:  cp from KDDC directory the file:  KDDC20160525_001527_V06.gz into the main directory.  Lvl2 files will be placed in a directory called “images”, and opaws2d files will be placed in a directory called “opaws_files”.  Plots of gridded superobbed fields (REF and VR) are placed in the main directory when created.

 

Example command lines for lvl2_plot.py:

 

1.    python lvl2_plot.py -f KDDC20160525_001527_V06.gz -q None -p 1 -u phase

a.    create plot of 6 variables (REF, VR, SPEC_W, Zdr, KDP, PHIDP), no QC, plot 2nd tilt level, phase unfolding

 

2.    python lvl2_plot.py --plot2 -f KDDC20160525_001527_V06.gz -q None -p 1 -u phase

a.    create plot of REF and VR, no DBZ QC, plot 2nd title, phase unfolding

 

3.    python lvl2_plot.py --plot2 -f KDDC20160525_001527_V06.gz -q MetSignal -p 1 -u phase

a.    create plot of REF and VR, MetSig QC, plot 2nd title, phase unfolding

 

Example command lines for opaws2d.py:

 

1.    python opaws2d.py -f KDDC20160525_001527_V06.gz -q None -p 1 –w -u phase

a.    no QC, plot 2nd tilt level, phase unfolding, -w to DART and netCDF files

 

2.    python opaws2d.py -f KDDC20160525_001527_V06.gz -q None -p 1 –w -u region

a.    no QC, plot 2nd tilt level, region unfolding, write out DART and netCDF files

 

3.    python opaws2d.py -f KDDC20160525_001527_V06.gz -q MetSignal -u phase

a.    MetSig QC, plot 2nd title, phase unfolding, write out DART and netCDF files

 

You can compare the outputs from your tests to the files in KDDC directory.

 

Main programs included

 

·      opaws2d.py:  program to read level-2 files directly, perform quality control and unfolding, and then thin the data to a co-plane Cartesian grid and write the superobbed data out into DART format.  See dictionaries at the top of the file for how the grid and output is setup.    

 

·      lvl2_plot.py:  program to plot 2 or 6 panel level-2 data at a certain tilt.  For the two panel plots, reflectivity and radial velocity are plotted.  For the 6 panel plots, spectrum width, cross-correlation coefficient, different reflectivity, and differential phase are also plotted.  See dictionaries at the top of the file for how the plotting/gridding parameters.    

 

·      mrms5_2_dart.py:   This program takes the NSSL MRMS systems’s 5 km gridded reflectivity (20 levels at 5 km spacing) and converts it to DART format.   This data is available from NSSL’s MRMS LDM feed.

 

·      mrms1_2_dart.py:  This program IS NOT YET written – it will take the operational 1 km MRMS feed from NCEP and thin the grid to the desired N km grid.

 

·      MetQC.ipyn:  Jupyter notebook which is used to test unfolding and QC codes.  Fun to play with.

 

Program needed for main programs

 

·      MetSignal.py:  is a python implementation of John Krause’s (2016, J. Atmos. Ocean. Tech) meteorological signal detection algorithm.  Using dual-pol data, the algorithm trys to determine where radar returns are associated with precipitation.  Good for removing ground clutter and turbines.  The version implemented is actually 2D, though differential phase remains 1D.  One should tune the threshold value for one’s application.

 

·      cressman.f90; fcompile_cress.f90:  fortran subroutine (and f2py compiler script) that will compile the cressman routine needed for opaws2d.py.

·      dart_tools.py:  python code that outputs the various radar fields into DART format.
